# Keli Client AI Rules

这些规则适用于 `keli-client`。根目录 `AGENTS.md` 的全局规则同样适用。

## 客户端职责

`keli-client` 是 Keli 的桌面和 Android 客户端，负责：

- 登录和会话保存。
- 用户信息展示。
- 节点列表、选择、收藏、筛选。
- 节点测速和失败原因展示。
- Windows/macOS/Android 代理连接。
- 商店套餐、订单、支付。
- 日志和诊断导出。

客户端不能自己发明业务流程或假数据。所有业务数据必须来自 `keliboard` API，商店和支付流程必须对照 `keli-user`。

## 平台能力规则

### Windows

- 可以使用本地 sing-box 进程。
- 可以使用 Clash API 做测速。
- 可以管理系统代理。
- 测速失败必须区分 API 未就绪、节点超时、配置缺出站、代理不存在。

### macOS

- UI 可以和 Windows 保持主结构一致。
- 代理核心、系统代理、权限模型必须单独确认，不要直接套 Windows 实现。

### Android

- Android 使用 VPNService。
- 云手机不能作为真实延迟基准。
- 如果移动端核心没有 Clash API，不应显示假测速。
- Android 的连接验证优先看：
  - APK 能否安装。
  - Activity 能否启动。
  - VPN 权限是否授权。
  - hiddify-core AAR 是否打包。
  - 配置是否写入。
  - VPN service 是否启动。

## 测速规则

禁止把所有失败都显示为 "超时"。

UI 必须能表达：

- `未测`：用户还没有触发测速。
- `测速中`：当前正在执行。
- `超时`：节点请求超时。
- `API未就绪`：Clash API 没启动或没响应。
- `无代理`：API 没返回该代理。
- `无出站`：配置缺少真实节点出站。
- `核心失败`：测速核心启动失败。
- `不可测`：当前平台不支持本地测速。
- `未授权`：Android VPN 未授权。
- `缺核心`：移动端核心未打包。

自动测速要求：

- 打开节点选择弹窗可以自动快测。
- 自动快测不能拖慢页面打开。
- 手动全部测速可以更完整、更慢。
- 批量测速优先复用核心，避免每个节点重复启动核心。

## 连接规则

连接流程必须按真实状态推进：

1. 拉取节点 sing-box 配置。
2. 写入本地配置。
3. 准备核心或 VPN 权限。
4. 启动核心或 VPN service。
5. 更新连接状态和运行统计。
6. 失败时记录可读错误。

禁止：

- 核心未真正启动就显示已连接。
- Android VPN 未授权时继续伪装连接。
- 浏览器没网时只显示泛化失败，不记录核心状态。

## 商店规则

商店必须对照 `keli-user` 和 `keliboard`。

必须支持：

- 获取真实套餐。
- 周期选择。
- 升级套餐。
- 续费套餐。
- 流量重置。
- 创建订单。
- 已有未支付订单时阻止重复创建。
- 支付方式选择。
- 外部支付跳转。
- 站内二维码支付。
- 支付后刷新订单和用户套餐。

禁止：

- 假套餐。
- 假价格。
- 重复展示套餐信息。
- 展示自己的当前套餐卡片作为商品。
- 只做表面 UI，不接真实 API。

## UI 规则

桌面端：

- 首页要紧凑，不留大块无意义空白。
- 个人信息不显示 UUID。
- 当前节点区域可以选择节点和测速。
- 代理模式区域不能空，要显示与实际平台相关的能力。
- 日志必须可读，错误要尽量归类。

移动端：

- 不直接缩小桌面布局。
- 底部导航、节点列表、商店流程要适配窄屏。
- 弹窗和购买流程必须适合触控。
- 不依赖 hover。

通用：

- UI 不能只好看，还要完成真实交互。
- 文案短而准确。
- 不展示用户无法理解或无法操作的技术字段。
- 状态标签要和真实状态一致。

## 开发验证

改 Dart/Flutter 后运行：

```powershell
flutter analyze
flutter test
```

改 Android 原生代码后尽量运行：

```powershell
flutter build apk --debug
```

如果本地沙箱不能运行 Flutter/Dart 子进程，必须说明原因，并在可用环境中重跑。

## 提交前检查

提交前确认：

- 没有无关格式化。
- 没有假数据残留。
- 没有把所有测速失败归为超时。
- 商店按钮不是空操作。
- 支付流程能刷新订单。
- Android 和桌面平台能力没有混用。
- 测试或静态检查已运行，或说明未运行原因。
